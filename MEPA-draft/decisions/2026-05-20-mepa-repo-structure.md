# Decision: Establish MEPA as a documentation-first operating repo

## Date
2026-05-20

## Status
Accepted

## Context
MEPA has been created as a new repository with prompt/context files, but it does not yet have a stable operating structure. Marshall Epie wants it to become the internal system for managing projects and eventually evolve into an app with a dashboard.

## Decision
MEPA will start as a documentation-first operations repository with clear directories for:
- portfolio-level docs
- per-project records
- prompts
- decisions
- active tasks
- reusable templates

The dashboard/app will be built after the documentation model and operating rules are proven in practice.

## Consequences
### Positive
- Immediate structure without overbuilding
- Easier AI continuity
- Lower risk of chaotic prompt sprawl
- Better foundation for a future app/data model

### Trade-offs
- Requires discipline in keeping docs current
- Dashboard capabilities will come after the operating model is stabilized
