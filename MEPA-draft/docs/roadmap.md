# Roadmap

## Phase 1 — Documentation foundation

Goal: make MEPA usable immediately as an operating repo.

Deliverables:
- portfolio map
- project status board
- per-project files
- operating rules
- task and decision templates

## Phase 2 — Structured operations

Goal: make the documentation easy to maintain and query.

Deliverables:
- normalized frontmatter/templates
- repeatable project update format
- cleaner decision log discipline
- possible JSON/YAML export layer for app consumption

## Phase 3 — Internal dashboard MVP

Goal: create a private dashboard for interacting with MEPA.

Possible MVP capabilities:
- project cards with status, blockers, and next action
- task board for active priorities
- decision log viewer
- links to product repos and deployments
- AI interaction panel for operational commands and summaries

## Phase 4 — Automation layer

Goal: make MEPA operationally proactive.

Possible capabilities:
- deployment/status checks
- portfolio summary generation
- issue ingestion from product repos
- recurring reviews and weekly reports
- AI-generated next-step suggestions
